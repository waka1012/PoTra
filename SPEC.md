# PoTra 仕様書

## 概要

**PoTra**（Podcast Transcriber）は、ポッドキャストのRSSフィードからエピソードをダウンロードし、MLX-Whisperで文字起こしするデスクトップGUIツール。  
ローカルの音声ファイルを直接指定して文字起こしすることも可能。

---

## 実行環境

- macOS（Apple Silicon: M1/M2/M3）
- Python 3.10 以上
- 依存ライブラリ：
  ```
  mlx-whisper
  feedparser
  requests
  ```
- 標準ライブラリ：`tkinter`, `threading`, `queue`, `logging`, `pathlib`, `datetime`

## タブ1：📡 RSSポッドキャスト

### 構成要素（上から順）

#### 1. RSS フィード URL 入力欄
- ラベルフレーム「RSS フィード URL」
- 1行テキスト入力フィールド（横幅いっぱい）

#### 2. キーワードフィルタ
- ラベルフレーム「キーワードフィルタ（空欄で全件）」
- テキスト入力フィールド（幅40文字）
- 「🔍 検索」ボタン
- 右端に件数ラベル（例: `16 件  キーワード: "西郷隆盛編"`）

#### 3. エピソード一覧
- ラベルフレーム「エピソード一覧」
- 上部に「全選択」「全解除」ボタン
- スクロール可能なチェックボックスリスト
  - Canvas + Scrollbar + 内部Frame の構成
  - 各エピソードを `ttk.Checkbutton` で1行ずつ表示
  - デフォルトはすべてチェック済み
  - Canvas幅に合わせてチェックボックスの幅を追従させる
- **並び順**: エピソード名の自然順ソート（昇順）
  - 数字部分を数値として比較（例: 1→2→…→9→10→11）
  - `re.split(r"(\d+)", title)` で分割し、数字部分を `int` 変換して比較

#### 4. 出力先フォルダ
- ラベル「出力先フォルダ:」
- テキスト入力フィールド（デフォルト: `~/potra_transcripts`）
- 「選択…」ボタン（フォルダ選択ダイアログ）

## タブ2：📁 ローカルファイル

### 構成要素（上から順）

#### 1. ファイル追加
- ラベルフレーム「ファイル・フォルダ追加」
- 「📄 MP3ファイルを選択（複数可）」ボタン
  - 対応拡張子: `.mp3`, `.m4a`, `.wav`, `.flac`, `.aac`, `.ogg`
  - 複数選択可
- 「📁 フォルダを選択」ボタン
  - 選択フォルダ内の `*.mp3` を再帰せず一覧に追加

#### 2. ファイル一覧
- ラベルフレーム「ファイル一覧」
- Listbox（複数選択モード `EXTENDED`、スクロール付き）
  - 表示内容: ファイル名のみ（フルパスではない）
- 「選択行を削除」ボタン（Listboxで選択した行を削除）
- 「リストをクリア」ボタン（全件削除）
- 右端に件数ラベル（例: `5 件`）

#### 3. 出力先フォルダ
- タブ1と同構成（変数は別）

## 共通コントロールエリア（タブ下部・全体）

### モデル選択
- ラベル「モデル:」
- `ttk.Combobox`（読み取り専用、幅28文字）
- 選択肢：
  | 表示名 | モデルパス |
  |---|---|
  | `large-v3（高精度）` | `mlx-community/whisper-large-v3-mlx` |
  | `large-v3-turbo（高速）` | `mlx-community/whisper-large-v3-turbo` |
- デフォルト: `large-v3（高精度）`

### 開始・中断ボタン
- 「▶ 開始」ボタン
  - 処理中は `state=DISABLED`
- 「⏹ 中断」ボタン
  - 待機中は `state=DISABLED`、処理中は `NORMAL`
  - フラグ `stop_requested = True` を立てるのみ（強制終了ではなく、現在処理中のファイルが終わり次第停止）

### ステータスラベル
- 処理中: `(3/16) エピソードタイトル`
- 完了時: `完了: 14 件 / スキップ: 2 件 / エラー: 0 件`
- 初期値: `待機中`

### プログレスバー
- `ttk.Progressbar`、最大値100
- 各ファイル処理開始時に `(i-1)/total * 100` を設定
- 全件完了時に `100` を設定

### ログエリア
- `scrolledtext.ScrolledText`
- 高さ9行、`Courier New` 10pt、`state=DISABLED`（直接編集不可）
- 新規メッセージ追加時に末尾へ自動スクロール
- フォーマット: `[HH:MM:SS] メッセージ`

## 処理フロー

### 開始ボタン押下時

1. アクティブなタブを判定（`notebook.index(notebook.select())`）
2. タブ0（RSS）の場合
   - チェック済みエピソードを `[(title, url), ...]` で収集
   - 0件なら警告ダイアログを出して中止
3. タブ1（ローカル）の場合
   - `self.local_files` を `[(stem, filepath), ...]` に変換
   - 0件なら警告ダイアログを出して中止
4. `mlx_whisper` が未インストールならエラーダイアログを出して中止
5. 出力先フォルダを作成（`mkdir(parents=True, exist_ok=True)`）
6. `stop_requested = False` にリセット
7. バックグラウンドスレッドで `_run_task()` を起動

### バックグラウンド処理（`_run_task`）

各アイテムについて以下を順に実行。`stop_requested` が True なら即座に中断。

#### ステップ1：スキップ判定
- 出力ファイル（`{安全なファイル名}.md`）が既存なら **スキップ**してログに記録

#### ステップ2：ダウンロード（RSSモードのみ）
- MP3ファイルが未存在の場合のみダウンロード
- `requests.get(url, stream=True, timeout=120)`
- チャンクサイズ: `65536` bytes
- ダウンロード完了時にファイルサイズ（MB）をログ出力

#### ステップ3：文字起こし
- `mlx_whisper.transcribe(audio_file, path_or_hf_repo=model_path, language="ja", word_timestamps=False)`
- `result["segments"]` を Markdown 形式に変換してファイルに書き込み（UTF-8）
- segments が空の場合は `result["text"]` にフォールバック

#### 出力フォーマット（Markdown）

Whisperの書き起こしテキストから話者名を自動検出し、`**Name**: content` 形式で出力する。  
話者が検出できない場合はタイムスタンプ形式にフォールバックする。

**話者検出ロジック**
- 正規表現 `([一-龯ぁ-んァ-ヶa-zA-Z]{2,6})\s(?=\S)` で候補語を抽出
- 3回以上出現する候補を話者名とみなす
- 話者名が検出された場合（例: 樋口、深井）:

```markdown
世界の歴史キュレーションプログラム コテンラジオ...（冒頭テキスト）

**樋口**: このラジオは歴史を愛し...

**深井**: はい、同じ株式会社コテンの...
```

- 話者名が検出できない場合（フォールバック）:

```markdown
- [00:00:05] テキスト内容...
- [00:01:23] 次のテキスト内容...
```

#### ファイル名の生成ルール
- `title` から `\ / : * ? " < > |` を除去
- 先頭80文字に切り詰め
- 前後の空白を除去
- 拡張子は `.md`
- 例: `【45-1】西郷隆盛編①〜薩摩の風雲児〜.md`

#### ログ出力（各ステップ）
```

[10:23:01] 処理開始  全 16 件 | モデル: mlx-community/...
[10:23:01] 出力先: /Users/yuki/potra_transcripts
[10:23:01] ==================================================
[10:23:01]
[10:23:01] [1/16] 【45-1】西郷隆盛編①〜薩摩の風雲児〜
[10:23:01]   ダウンロード中…
```

## ロギング仕様

### GUIログ（`log_queue` 経由）
- バックグラウンドスレッドから直接 `log_area` を操作しない
- `queue.Queue` にメッセージを put し、`root.after(100, _poll_log_queue)` でポーリング
- 100ms ごとにキューを drain してログエリアに追記

### ファイルログ
- 保存先: `~/potra_logs/potra_YYYYMMDD_HHMMSS.log`
- フォーマット: `%(asctime)s [%(levelname)s] %(message)s`
- レベル: DEBUG（ファイル）、INFO/ERROR（GUIと同等）
- アプリ起動時に1ファイル生成（セッション単位）

## 起動時チェック

依存ライブラリの import を試み、失敗したものを `showwarning` ダイアログで一覧表示する。  
アプリ自体は起動する（未インストールの機能を使おうとしたときに個別にエラー表示）。

チェック対象:
- `feedparser`（RSSタブで使用）
- `requests`（RSSダウンロードで使用）
- `mlx_whisper`（文字起こしで使用）

---

## 注意事項・制約

- `mlx_whisper.transcribe()` はブロッキング呼び出しのため、必ずバックグラウンドスレッドで実行する
- GUIの更新は必ず `root.after(0, lambda: ...)` を経由する
- 中断は「現在のファイル処理完了後に停止」であり、即時強制終了ではない
- RSSダウンロード済みMP3は再ダウンロードしない（ファイル存在チェック）
- 文字起こし済みMDは再処理しない（スキップ）
- これらのスキップ挙動により、途中中断・再実行が安全に行える

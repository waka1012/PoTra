# PoTra 仕様書

## 概要

**PoTra**（Podcast Transcriber）は、ポッドキャストのRSSフィードからエピソードをダウンロードし、MLX-Whisperで文字起こしするデスクトップGUIツール。  
ローカルの音声ファイルを直接指定して文字起こしすることも可能。

---

## 実行環境

- macOS（Apple Silicon: M1/M2/M3）
- Python 3.10 以上
- 依存ライブラリ：
  ```
  mlx-whisper
  feedparser
  requests
  ```
- 標準ライブラリ：`tkinter`, `threading`, `queue`, `logging`, `pathlib`, `datetime`

## タブ1：📡 RSSポッドキャスト

### 構成要素（上から順）

#### 1. RSS フィード URL 入力欄
- ラベルフレーム「RSS フィード URL」
- 1行テキスト入力フィールド（横幅いっぱい）

#### 2. キーワードフィルタ
- ラベルフレーム「キーワードフィルタ（空欄で全件）」
- テキスト入力フィールド（幅40文字）
- 「🔍 検索」ボタン
- 右端に件数ラベル（例: `16 件  キーワード: "西郷隆盛編"`）

#### 3. エピソード一覧
- ラベルフレーム「エピソード一覧」
- 上部に「全選択」「全解除」ボタン
- スクロール可能なチェックボックスリスト
  - Canvas + Scrollbar + 内部Frame の構成
  - 各エピソードを `ttk.Checkbutton` で1行ずつ表示
  - デフォルトはすべてチェック済み
  - Canvas幅に合わせてチェックボックスの幅を追従させる
- **並び順**: エピソード名の自然順ソート（昇順）
  - 数字部分を数値として比較（例: 1→2→…→9→10→11）
  - `re.split(r"(\d+)", title)` で分割し、数字部分を `int` 変換して比較

#### 4. 出力先フォルダ
- ラベル「出力先フォルダ:」
- テキスト入力フィールド（デフォルト: `~/potra_transcripts`）
- 「選択…」ボタン（フォルダ選択ダイアログ）

## 共通コントロールエリア（タブ下部・全体）

### モデル選択
- ラベル「モデル:」
- `ttk.Combobox`（読み取り専用、幅28文字）
- 選択肢：
  | 表示名 | モデルパス |
  |---|---|
  | `large-v3（高精度）` | `mlx-community/whisper-large-v3-mlx` |
  | `large-v3-turbo（高速）` | `mlx-community/whisper-large-v3-turbo` |
- デフォルト: `large-v3（高精度）`

### 語彙ファイル選択
- ラベル「語彙:」
- `ttk.Combobox`（読み取り専用、幅24文字）
- 選択肢: 「なし」＋ `vocabularies/` 内の `.txt` ファイル名一覧（起動時に読み込み）
- 選択時にトークン数をカウントしてラベルに表示
  - 200トークン以下: 緑で `✅ XX トークン / 200`
  - 200トークン超: 黄色で `⚠️ XX トークン / 200 — 超過分は無視されます`
  - 動作はブロックしない（警告のみ）
- トークンカウントには tiktoken（`cl100k_base` エンコーディング）を使用

### 語彙ファイル仕様（`vocabularies/*.txt`）
- 1行1単語
- `#` で始まる行はコメントとして無視
- ファイルが存在しない・ディレクトリが空の場合は `initial_prompt` なしで動作

### 開始・中断ボタン
- 「▶ 開始」ボタン
  - 処理中は `state=DISABLED`
- 「⏹ 中断」ボタン
  - 待機中は `state=DISABLED`、処理中は `NORMAL`
  - 押下時: `stop_requested = True` をセット、ボタンを即座に `DISABLED`、ステータスを「中断中…」に更新
  - 強制終了ではなく、ダウンロード中はチャンク間で停止、文字起こし中は完了後に停止

### ステータスラベル
- 処理中: `(3/16) エピソードタイトル`
- 中断操作後: `中断中…（現在の処理が終わり次第停止）`
- 完了時: `完了: 14 件 / スキップ: 2 件 / エラー: 0 件`
- 初期値: `待機中`

### プログレスバー
- `ttk.Progressbar`、最大値100
- 各ファイル処理開始時に `(i-1)/total * 100` を設定
- 全件完了時に `100` を設定

### ログエリア
- `scrolledtext.ScrolledText`
- 高さ9行、`Courier New` 10pt、`state=DISABLED`（直接編集不可）
- 新規メッセージ追加時に末尾へ自動スクロール
- フォーマット: `[HH:MM:SS] メッセージ`

## 処理フロー

### 開始ボタン押下時

1. アクティブなタブを判定（`notebook.index(notebook.select())`）
2. タブ0（RSS）の場合
   - チェック済みエピソードを `[(title, url), ...]` で収集
   - 0件なら警告ダイアログを出して中止
3. タブ1（ローカル）の場合
   - `self.local_files` を `[(stem, filepath), ...]` に変換
   - 0件なら警告ダイアログを出して中止
4. `mlx_whisper` が未インストールならエラーダイアログを出して中止
5. 出力先フォルダを作成（`mkdir(parents=True, exist_ok=True)`）
6. `stop_requested = False` にリセット
7. バックグラウンドスレッドで `_run_task()` を起動

### バックグラウンド処理（`_run_task`）

各アイテムについて以下を順に実行。`stop_requested` が True なら即座に中断。

#### ステップ1：スキップ判定
- 出力ファイル（`{安全なファイル名}.md`）が既存なら **スキップ**してログに記録

#### ステップ2：ダウンロード（RSSモードのみ）
- MP3ファイルが未存在の場合のみダウンロード
- `requests.get(url, stream=True, timeout=120)`
- チャンクサイズ: `65536` bytes
- 書き込み先は一時ファイル（`.mp3.tmp`）。完了後に `.mp3` にリネーム
- チャンクループ内で `stop_requested` を確認し、中断時は一時ファイルを削除して即停止
- ダウンロード完了時にファイルサイズ（MB）をログ出力

#### ステップ3：文字起こし
- 語彙ファイルが選択されている場合、単語をカンマ区切りで結合して `initial_prompt` を生成
  - 200トークン超の場合は末尾の単語から切り捨て（tiktoken `cl100k_base` で計算）
  - tiktoken 未インストール時は切り捨てなしで全単語を使用
- `initial_prompt` をログに出力（設定されている場合）
- `mlx_whisper.transcribe(audio_file, path_or_hf_repo=model_path, language="ja", word_timestamps=False, initial_prompt=...)`
- 結果を `formatter` 関数に渡し、戻り値をファイルに書き込み（UTF-8）
- `formatter` は `formatter.py` の `custom_formatter`。`None` の場合は `default_formatter` を使用

#### デフォルト出力フォーマット（Markdown）

セグメントごとにタイムスタンプを付与し、話者名を自動検出して出力する。

**話者検出ロジック**
- 正規表現 `([一-龯ァ-ヶ][一-龯ぁ-んァ-ヶ]{0,4})\s(?=\S)` で候補語を抽出
  - 漢字またはカタカナで始まる語のみを対象にすることで、前の発言末尾との混入を防ぐ
- 3回以上出現する候補を話者名とみなす
- 話者名が検出された場合:

```markdown
- [00:00:05] **田中**: 今日のテーマについて教えてもらえますか？
- [00:00:12] **鈴木**: はい、生成AIの最新動向についてお話しします。
- [00:00:20] **鈴木**: 特にマルチモーダルの進化が面白いですね。
```

- 話者切り替えが検出されないセグメントは直前の話者を引き継ぐ
- **同一話者の連続セグメントはマージして1行にまとめる**
  - テキストはスペースで結合し、タイムスタンプは連続セグメントの先頭を使用
  - 話者が変わったタイミングで新しい行を開始する
- 話者名が検出できない場合（フォールバック）:

```markdown
- [00:00:05] テキスト内容...
- [00:01:23] 次のテキスト内容...
```

#### 出力フォーマットのカスタマイズ

`formatter.py` の `custom_formatter` に関数を設定することで出力形式を変更できる。

```python
# formatter.py
def custom_formatter(result: dict) -> str:
    # result["text"]               : 全文テキスト
    # result["segments"][n]["start"]: 開始時刻（秒）
    # result["segments"][n]["end"]  : 終了時刻（秒）
    # result["segments"][n]["text"] : テキスト
    return result["text"].strip()

custom_formatter = custom_formatter  # 有効化
```

`custom_formatter = None`（デフォルト）の場合は `worker.py` の `default_formatter` が使われる。

#### ファイル名の生成ルール
- `title` から `\ / : * ? " < > |` を除去
- 先頭80文字に切り詰め
- 前後の空白を除去
- 拡張子は `.md`
- 例: `【45-1】西郷隆盛編①〜薩摩の風雲児〜.md`

## ロギング仕様

### GUIログ（`log_queue` 経由）
- バックグラウンドスレッドから直接 `log_area` を操作しない
- `queue.Queue` にメッセージを put し、`root.after(100, _poll_log_queue)` でポーリング
- 100ms ごとにキューを drain してログエリアに追記

### ファイルログ
- 保存先: `~/potra_logs/potra_YYYYMMDD_HHMMSS.log`
- フォーマット: `%(asctime)s [%(levelname)s] %(message)s`
- レベル: DEBUG（ファイル）、INFO/ERROR（GUIと同等）
- アプリ起動時に1ファイル生成（セッション単位）

## 多言語対応（i18n）

### 仕組み
- GUI に表示するすべてのテキストは `locale/<lang>.json` で管理する
- アプリ起動時に `config.json` の `"language"` キーを読み込み、対応する JSON を読む
- JSON が存在しない場合は `locale/en.json`（英語）にフォールバックする
- `worker.py` の `run_task` も `messages=` でロケール辞書を受け取り、ログ文字列を切り替える

### 設定ファイル
`config.json`（デフォルト: 英語）:
```json
{ "language": "en" }
```

日本語に切り替えるには:
```json
{ "language": "ja" }
```

### ロケールファイルの構造
`locale/en.json` と `locale/ja.json` に同一キーセットを持たせる。  
新言語を追加する場合は同名のキーを持つ JSON を `locale/` に置き、`config.json` を変更する。

## 注意事項・制約

- `mlx_whisper.transcribe()` はブロッキング呼び出しのため、必ずバックグラウンドスレッドで実行する
- GUIの更新は必ず `root.after(0, lambda: ...)` を経由する
- 中断ボタン押下時はステータスを即座に「中断中…」に更新し、ボタンを無効化する
- ダウンロード中の中断はチャンクループ内で検出し即停止（途中ファイルは削除）
- 文字起こし中の中断は `mlx_whisper.transcribe()` がブロッキングのため完了後に停止
- RSSダウンロード済みMP3は再ダウンロードしない（ファイル存在チェック）
- 文字起こし済みMDは再処理しない（スキップ）
- これらのスキップ挙動により、途中中断・再実行が安全に行える
